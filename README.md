# responder-httpbin: HTTP Request &amp; Response Service

A [Kenneth Reitz](http://kennethreitz.org/) project.

**responder-httpbin** is the classic [httpbin](https://github.com/postmanlabs/httpbin)
API, re-implemented on top of the [**responder**](https://responder.kennethreitz.org/)
web framework (v5). It's a dogfooding fork — a faithful, drop-in port of every
httpbin endpoint — built to exercise responder end to end.

> This is **not** httpbin.org. The original httpbin was sold to Postman Labs years
> ago; this fork exists to put responder through its paces.

## Why?

`httpbin` is a wonderfully thorough exercise for a web framework: streaming,
async delays, content negotiation, cookies, digest auth, range requests,
gzip/deflate/brotli, conditional requests, and more. Re-implementing it on
responder is a way to make sure the framework handles all of it cleanly — and
shows off responder's batteries-included features:

- **Native OpenAPI + Swagger UI** — the interactive docs at `/` and the schema at
  `/schema.yml` are generated by responder (no flasgger).
- **ASGI middleware** — CORS, chunked-request handling, and OPTIONS preflight are
  one small `HttpbinMiddleware`.
- **Async streaming** — `/stream`, `/stream-bytes`, `/drip`, `/delay`, and `/range`
  use `@resp.stream` and `await asyncio.sleep`.
- **Typed path params, templates, and the built-in httpx test client.**

## Run locally

With Docker (uses [granian](https://github.com/emmett-framework/granian), a Rust
ASGI server, in production):

```sh
docker build -t responder-httpbin .
docker run -p 80:80 responder-httpbin
```

From source with [uv](https://docs.astral.sh/uv/):

```sh
uv sync
uv run granian --interface asgi --port 5000 httpbin:api   # production server
# or, for a quick dev server (responder's built-in, with reload):
uv run python -m httpbin.core
```

Then open <http://localhost:5000/> for the interactive Swagger docs.

## Tests

The full httpbin test suite is ported to responder's built-in test client:

```sh
uv run pytest test_httpbin.py
```
