# responder-httpbin: HTTP Request &amp; Response Service

[![test](https://github.com/kennethreitz/responder-httpbin/actions/workflows/test.yml/badge.svg)](https://github.com/kennethreitz/responder-httpbin/actions/workflows/test.yml)

A [Kenneth Reitz](http://kennethreitz.org/) project.

**responder-httpbin** is the classic [httpbin](https://github.com/postmanlabs/httpbin)
API, re-implemented on top of the [**responder**](https://responder.kennethreitz.org/)
web framework (v5). It's a dogfooding fork — a faithful, drop-in port of every
httpbin endpoint — built to exercise responder end to end.

> This is **not** httpbin.org. The original httpbin was sold to Postman Labs years
> ago; this fork exists to put responder through its paces.

## Why?

`httpbin` is a wonderfully thorough exercise for a web framework: streaming,
async delays, content negotiation, cookies, digest auth, range requests,
gzip/deflate/brotli, conditional requests, and more. Re-implementing it on
responder is a way to make sure the framework handles all of it cleanly — and to
show off responder's batteries-included features:

- **Native OpenAPI + Swagger UI** — the interactive docs at `/` and the schema at
  `/schema.yml` are generated by responder (no flasgger).
- **ASGI middleware** — CORS, chunked-request handling, and OPTIONS preflight are
  one small `HttpbinMiddleware`.
- **Async streaming** — `/stream`, `/stream-bytes`, `/drip`, `/delay`, and `/range`
  use `@resp.stream` and `await asyncio.sleep`.
- **Typed path params, Jinja templates, and the built-in httpx test client.**

The whole port is ~1,800 lines and passes a 74-test suite (the original httpbin
tests, ported to responder's test client, plus parity regressions).

## Run locally

### With Docker

The image installs dependencies with [uv](https://docs.astral.sh/uv/) and serves
the app with [granian](https://github.com/emmett-framework/granian), a fast Rust
ASGI server:

```sh
docker build -t responder-httpbin .
docker run -p 80:80 responder-httpbin
# or
docker compose up --build
```

Then open <http://localhost/> for the interactive Swagger docs.

### From source

```sh
uv sync
uv run granian --interface asgi --port 5000 httpbin:api   # production server
# or, for a quick dev server (responder's built-in):
uv run python -m httpbin.core
```

Then open <http://localhost:5000/>.

> **Sessions:** set `SECRET_KEY` in production so signed session cookies survive
> restarts and are shared across workers.

## Tests

The full httpbin test suite is ported to responder's built-in test client:

```sh
uv run pytest test_httpbin.py
```

## Endpoints

Interactive docs live at `/` (Swagger UI); the OpenAPI schema is at `/schema.yml`.

| Group | Endpoints |
| --- | --- |
| **HTTP methods** | `/get` · `/post` · `/put` · `/patch` · `/delete` |
| **Auth** | `/basic-auth/{user}/{passwd}` · `/hidden-basic-auth/{user}/{passwd}` · `/bearer` · `/digest-auth/{qop}/{user}/{passwd}[/{algorithm}[/{stale_after}]]` |
| **Status codes** | `/status/{codes}` (single, or weighted random e.g. `200:3,500:1`) |
| **Request inspection** | `/headers` · `/ip` · `/user-agent` |
| **Response inspection** | `/cache` · `/cache/{n}` · `/etag/{etag}` · `/response-headers` |
| **Response formats** | `/gzip` · `/deflate` · `/brotli` · `/html` · `/xml` · `/json` · `/robots.txt` · `/deny` · `/encoding/utf8` |
| **Dynamic data** | `/uuid` · `/base64/{value}` · `/bytes/{n}` · `/stream-bytes/{n}` · `/stream/{n}` · `/delay/{n}` · `/drip` · `/range/{n}` · `/links/{n}` |
| **Cookies** | `/cookies` · `/cookies/set` · `/cookies/set/{name}/{value}` · `/cookies/delete` |
| **Images** | `/image` (Accept-negotiated) · `/image/png` · `/image/jpeg` · `/image/webp` · `/image/svg` |
| **Redirects** | `/redirect/{n}` · `/redirect-to?url=` · `/relative-redirect/{n}` · `/absolute-redirect/{n}` |
| **Anything** | `/anything` · `/anything/{anything}` (echoes whatever you send) |

A classic man-page-style index is also served at `/legacy`.
